## 【概要説明】AWS ECSを利用したBlue/Greenデプロイメント環境の構築について

### 1. このプロジェクトの目的
システムのアップデート時に「サービスを止めない（ダウンタイムゼロ）」かつ「安全に切り替える」環境を、AWSのコード管理ツール（CDK）を用いて構築しました。



### 2. システム構成の全体像
今回構築したのは、**「同じ中身のシステムを2セット（BlueとGreen）用意し、入り口のスイッチで切り替える」**という構成です。

* **VPC（ネットワーク）**: セキュリティを考慮し、システム本体は「プライベート空間」に配置。ECR（画像倉庫）との通信を安定させるための専用道路（VPCエンドポイント）も整備済みです。
* **ECR（イメージ倉庫）**: Webサーバーの「中身」を保存する場所。
    * `latest` タグ：青色の背景が出る「Blue用」イメージ
    * `green` タグ：緑色の背景が出る「Green用」イメージ
* **NLB（ロードバランサー）**: ユーザーからのアクセスを、BlueまたはGreenのサービスへ振り分ける「交通整理役」です。

### 3. 【重要】CDKによるデプロイの仕組み（ここが要です！）
このシステムの最大の特徴は、**CDK（Infrastructure as Code）を使って、ネットワーク設定からアプリの切り替えまでを一括管理している点**です。

#### ① リスナーによる3つの入り口
NLBには、以下の3つのアクセスポートを設定しています。
* **Port 80 (本番用)**: `active_color` 設定により、BlueかGreenのどちらかに繋がります。
* **Port 8000 (Blue検証用)**: 常にBlue環境に繋がります。
* **Port 8080 (Green検証用)**: 常にGreen環境に繋がります。

#### ② デプロイ時の「スイッチ」の仕組み
`cdk.json` という設定ファイルにある **`active_color`** という値が、本番（Port 80）の「切り替えスイッチ」になっています。

```json
// cdk.json のイメージ
"active_color": "blue" // これを "green" に書き換えてデプロイすると...
```

1.  **設定の変更**: 管理者が `active_color` を書き換えます。
2.  **CDKデプロイ実行**: `npx cdk deploy` を叩くと、AWS CloudFormationが動き出します。
3.  **瞬時の切り替え**: CloudFormationは、NLBの「Port 80」の接続先ターゲットを、BlueからGreenへ**一瞬で**書き換えます。



### 4. 実際の運用フロー
チームで開発を進める際の手順は以下のようになります。

1.  **新しいイメージの準備**: 
    修正したプログラムをビルドし、待機中（本番じゃない方）のタグを付けてECRにプッシュします。
2.  **プレビュー確認**: 
    検証用ポート（8000 or 8080）にアクセスし、新機能が正しく動いているか、色が正しく出ているかを確認します。この時、本番ユーザーには一切影響がありません。
3.  **本番切り替え（リリース）**: 
    確認が取れたら、CDKの `active_color` を書き換えてデプロイ。本番のアクセス先が数秒で新しい環境へ切り替わります。
4.  **万が一の切り戻し**: 
    もし本番で不具合が見つかったら、`active_color` を元の色に戻して再デプロイするだけで、即座に以前のバージョンへ復旧できます。

---

### 補足：トラブルシューティング済みの点
構築の過程で、以下の「ハマりポイント」を解決済みです。
* **ネットワークの疎通**: ECRからイメージを引く際のタイムアウト問題を、VPCエンドポイントの導入により解消しました。
